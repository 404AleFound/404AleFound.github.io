Kotlin Class

2026-05-16 · updated 2026-05-19 · programming-languages , kotlin-note

Note

与其他面向对象语言一样，Kotlin 使用类来封装数据和行为，以编写可复用的结构化代码。

类是对象的蓝图或模板，你通过构造函数来创建对象。当你创建类的一个实例时，就是基于该蓝图创建一个具体的对象。

1 常规类

类的内容包括，类头和类体：

  • 类头包括但不限于：
    • class 关键字
    • 类名
    • 类型参数（如有）
    • 主构造函数（可选）
  • 类体，也是可选的，由花括号 {} 包围，包含类成员：
    • 次构造函数
    • 初始化代码块
    • 函数
    • 属性
    • 嵌套类和内部类
    • 对象声明

1.7 抽象类

在 Kotlin 中，抽象类是指不能直接实例化的类。它们被设计为被子类继承，子类定义它们的具体行为。这种行为称为实现：

  • 抽象类可以声明抽象属性和函数，这些属性和函数必须在子类中实现。
  • 抽象类的构造函数用于定义子类必须满足的参数和初始化要求。
  • 一个抽象类可以同时拥有抽象和非抽象成员（属性和函数）。
  • 要声明一个成员为抽象，你必须显式使用 abstract 关键字。

2 数据类

Kotlin 中的数据类主要用于存储数据。对于每个数据类，编译器会自动生成额外的成员函数，这些函数允许你将实例打印为可读输出、比较实例、复制实例等。数据类使用 data 标记

数据类必须满足如下要求：

  • 主构造函数必须至少有一个参数
  • 所有主构造函数参数必须标记为 val 或 var
  • 数据类不能用关键字 Abstract、Open、Sealed或 Inner

编译器自动从主构造器中声明的所有属性派生以下成员：

  • equals() / hashCode()
  • toString()
  • componentN() 函数，对应于属性的声明顺序
  •  copy() 函数

此外，数据类成员的生成遵循以下关于成员继承的规则：

  • 如果数据类体中有 equals() 、 hashCode() 或 toString() 的显式实现，或者父类中有 final 实现，则不会生成这些函数，而是使用现有的实现。
  • 父类中有 componentN() 函数且它们是 open 并返回兼容类型，则将为数据类生成相应的函数并覆盖超类型的函数。如果由于签名不兼容或它们是 final 而无法覆盖超类型的函数，则会报告错误。
  • 不允许为 componentN() 和 copy() 函数提供显式实现。

4 扩展

Kotlin 关键字 Extention 允许您在不使用继承或装饰器等设计模式的情况下，为类或接口添加新功能。当您需要使用无法直接修改的第三方库时，它们非常有用。一旦创建，您就可以像调用原始类或接口的成员一样调用这些扩展。

注意 Extention 并不会改变原始类的定义，不会添加真正的成员，其只是一个语法糖，使得这个函数能够像 object.method() 的形式调用，其也无法访问类的 private/protected 成员。

综上，Extention 只是”语法层面的便利”，不是”结构层面的修改”。

6 继承

Note

在用类创建继承层次结构之前，请考虑使用抽象类或接口。您可以从抽象类和接口默认继承。它们的设计方式是让其他类可以继承它们的成员并实现它们。

Kotlin 中的所有类都有一个共同的超类 Any ，它是没有声明超类的类的默认超类.

Any 有三个方法： equals() 、 hashCode() 和 toString() 。因此，这些方法对所有 Kotlin 类都进行了定义。

默认情况下，Kotlin 类是 final 的——它们不能被继承。要使一个类可继承，用 open 关键字标记它：

6.1 open 关键字

在 Kotlin 中 open 关键字表示一个类或成员可以在子类中被覆盖。默认情况下， Kotlin 类及其成员是 final 的，这意味着它们不能被继承或覆盖。

如果重写了基类的成员，重写的成员默认也是公开的。如果想禁止类的子类被覆盖，可以显式地将重写的成员标记为 final 。

6.2 override 关键字

Kotlin 要求可覆盖成员和方法显示使用修饰符 override：

  • 子类重写父类成员时，override 关键字为必选项，缺省则编译报错。
  • 父类成员未标记 open 者，子类不得声明同签名成员。
  • open 对 final 类成员不产生实际效果。
  • 一个被标记为 override 的成员本身是开放的，因此在子类中可能会被覆盖，如果想要禁止重新覆盖，请使用 final。
