0912 排序数组

2026-04-22 · updated 2026-05-19 · leetcode

摘要：数组处理技巧题目，考察排序、二分查找或数组元素的巧妙操作。

Note

题目： 0912 排序数组

给你一个整数数组 nums，请你将该数组升序排列。你必须在不使用任何内置函数的情况下解决问题，时间复杂度为 O(nlog(n))，并且空间复杂度尽可能小。

题解：排序

该题可以使用常见的排序算法解决，常见的排序算法有：插入排序、希尔排序、选择排序、冒泡排序、归并排序、快速排序、堆排序、基数排序等，基本概念如下：

  • 时间复杂度： 算法执行所需的时间，通常用大 O 表示法描述最坏/最好/平均情况下的比较或移动次数。
  • 空间复杂度： 不含输入数据本身占用的空间，算法执行所需的额外辅助空间。
  • 稳定性： 大小相同的元素，若排序前后相对位置不变，则称该方法稳定；反之，则称该方法不稳定。稳定性只在有多个排序关键字时才有意义。

1 冒泡排序 Bubble Sort

核心思想： 通过相邻元素的比较和交换，让较大的元素逐渐 “浮” 到数组末端。

  • 时间复杂度：当待排序数组有序时，加标志位优化只需要一轮比较，时间复杂度 O(n)；当待排序数组完全逆序时，每轮都需要完整比较交换，时间复杂度 O(n^2) ；平均时间复杂度为 O(n^2)。
  • 空间复杂度：原地排序，仅需常数个临时变量。
  • 稳定性：稳定，相邻比较，相等时不交换。

2 选择排序 Selection Sort

核心思想： 每次从未排序部分选出最小（或最大）元素，放到已排序部分的末尾。

  • 时间复杂度：无论如何都要遍历所有剩余元素找最小值，因此时间复杂度总是为 O(n^2)。
  • 空间复杂度：O(1)，原地排序，仅需常数个临时变量。
  • 稳定性：不稳定，交换操作可能破坏相等元素的相对顺序。例如，对于数组 [5, 5, 1]，第一轮找到最小值 1 与第一个 5 交换，得到 [1, 5, 5]，两个 5 的相对顺序被交换破坏。

3 插入排序 Insertion Sort

核心思想： 将未排序元素逐个插入到已排序序列的合适位置。

  • 时间复杂度：最好时间复杂度，当已有序时，每个元素只需要比较一次，O(n)；最坏时间复杂度，当完全逆序，每个元素都要移动到最前面，O(n^2)；平均时间复杂度，O(n^2)。
  • 空间复杂度：O(1)，原地排序，仅需 key 临时变量。
  • 稳定性：稳定，从后往前比较，相等时停止（不越过相等元素）

4 希尔排序 Shell Sort

核心思想： 通过较大间隔的插入排序使数组”大致有序”，再逐步缩小间隔，直至间隔为 1 完成最终排序。

  • 时间复杂度：最好时间复杂度，取决于增量序列，O(nlogn)；最坏时间复杂度，O(n^2)；平均时间复杂度，O(nlogn) ~ O(n^2)。
  • 空间复杂度：O(1)，原地排序。
  • 稳定性：不稳定，分组插入排序可能打乱相等元素的相对顺序。

5 归并排序 Merge Sort

核心思想： 分治思想，将序列递归地分成两半分别排序，再将两个有序子序列合并成一个。

  • 时间复杂度：最好时间复杂度，总是将序列对半分割，O(nlogn)；最坏时间复杂度，与初始顺序无关，O(nlogn)；平均时间复杂度，O(nlogn)。
  • 空间复杂度：O(n)，需要与原始数组等长的辅助数组。
  • 稳定性：稳定，合并时相等元素优先取左半部分

6 快速排序 Quick Sort

核心思想： 通过基准元素将数组划分为左右部分，左边小于基准、右边大于基准，再递归处理左右子区间。

  • 时间复杂度：最好时间复杂度，每次划分均匀，递归深度不变，O(nlogn)；最坏时间复杂度，每次基准都是极值，O(n^2)；平均时间复杂度，O(nlogn)。
  • 空间复杂度：O(logn)，递归栈空间的平均深度。
  • 稳定性：不稳定，分区时的交换可能破坏相等元素顺序。

7 堆排序 Heap Sort

核心思想： 利用最大堆（或最小堆）数据结构，通过不断取出堆顶元素完成排序。

  • 时间复杂度：最好时间复杂度，建堆 O(n)，每次调整 O(logn)，总时间复杂度 O(nlogn)；最坏时间复杂度，与初始顺序无关，O(logn)；平均时间复杂度，O(nlogn)。
  • 空间复杂度：O(1)，原地排序，无需额外空间。
  • 稳定性：不稳定，堆调整时可能破坏相等元素的相对顺序。

选型建议

  • 数据量小（n < 1000）且基本有序：插入排序
  • 数据量中等且要求稳定：归并排序
  • 数据量大且不要求稳定：快速排序（实际最快）或 堆排序（无最坏退化风险）
  • 数据量极大需外存辅助：归并排序（多路归并）
  • 需要原地且最坏有保证：堆排序
